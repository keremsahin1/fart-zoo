## Commands

**Project generation** (required after adding or removing Swift files):
```bash
xcodegen generate
```

**Build:**
```bash
xcodebuild build -project FartZoo.xcodeproj -scheme "FartZoo Watch App" -destination 'platform=watchOS Simulator,id=7F7DEDA9-CCDB-494B-A572-B195DCC41C2F'
```

**Run all tests:**
```bash
xcodebuild test -project FartZoo.xcodeproj -scheme "FartZoo Watch App" -destination 'platform=watchOS Simulator,id=7F7DEDA9-CCDB-494B-A572-B195DCC41C2F'
```

**Run a single test class:**
```bash
xcodebuild test -project FartZoo.xcodeproj -scheme "FartZoo Watch App" -destination 'platform=watchOS Simulator,id=7F7DEDA9-CCDB-494B-A572-B195DCC41C2F' -only-testing:FartZooTests/QuestTests
```

**Target simulator:** Apple Watch Series 11 (42mm), ID: `7F7DEDA9-CCDB-494B-A572-B195DCC41C2F`

## Architecture

Standalone watchOS 10+ app (no paired iPhone app). Uses SwiftUI + SwiftData + CloudKit.

### Layer overview

```
FartZoo Watch App/
  Models/          — Value types and enums (not persisted)
  Data/            — AnimalDatabase singleton (static data)
  ViewModels/      — @Observable classes (one per major feature)
  Views/           — SwiftUI views
  Services/        — SoundManager singleton
```

### Data model split

There are two kinds of models with a sharp boundary between them:

- **Static definitions** (`AnimalDefinition`, `HybridAnimal` struct, `Rarity`, `WorldLocation`) — value types in `Models/Animal.swift`, never persisted. `AnimalDatabase.shared` is the single source for all 111 animals.
- **Persisted state** (`CollectedAnimal`, `CollectedHybrid`, `PlayerProgress`) — SwiftData `@Model` classes in `Models/PlayerProgress.swift`, synced via CloudKit (`iCloud.com.fartzoo.watchkitapp`).

Collected animals are stored by ID only (`animalID: String`) and looked up from `AnimalDatabase` at display time.

### ViewModels

All ViewModels use `@Observable` (not `ObservableObject`).

- `TeleportViewModel` — manages globe animation and animal list; calls into `AnimalDatabase` for location-filtered animals
- `QuestViewModel` — owns all three quest types (TAP/SPIN/TIMING) state machine, timers, coin deduction/reward; difficulty scales by `animal.rarity`
- `DailyChallengeViewModel` — initialized once in `RootView` and passed app-wide via `.environment(vm)`; tracks 5-of-15 rotating daily challenges using a bitmask on `PlayerProgress`

### App entry & environment

`FartZooApp` creates the SwiftData `ModelContainer` and warms the globe SceneKit scene in a background task. `RootView` lazily creates `PlayerProgress` (singleton pattern via `@Query` + `context.insert`) and initializes `DailyChallengeViewModel`, which is then passed down via `.environment()`.

### Key behaviors

- **Coin economy**: Catching costs `rarity.coinCost`, winning rewards `coinCost / 2`. Failing costs coins with no reward.
- **Quest difficulty**: All targets/timers are pure functions of `animal.rarity` — no separate config.
- **Daily challenges**: 5 challenges selected from a pool of 15 via `(dayOfYear - 1) % 15`, shifting by 1 each day. Progress tracked in-memory; completion persisted to `PlayerProgress` via bitmask.
- **Hybrid names**: Generated by `HybridAnimal.hybridName` — takes 70% prefix of parent1 + 60% suffix of parent2.
- **Sound**: Fart sounds are `.wav` files in `Resources/Sounds/`; haptics via `WKInterfaceDevice.current().play(...)`.

### Tests

Tests live in `FartZooTests/` and are organized by feature (one file per domain). Always add a regression test for every bug fix. Tests target watchOS and import `@testable import FartZoo_Watch_App`.
